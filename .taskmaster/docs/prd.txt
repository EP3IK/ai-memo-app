# Overview  
Memora는 AI 기반 웹 메모 애플리케이션으로, 사용자의 음성 또는 텍스트 입력을 받아 AI가 자동으로 요약 및 분류해주는 기능을 제공합니다. STT(Speech-to-Text) 기술을 통해 빠른 음성 메모 작성이 가능하며, 모든 데이터는 웹 애플리케이션의 데이터베이스에 저장됩니다. 기존 메모 서비스의 단순 저장 기능을 넘어서, 시간이 지날수록 중요한 정보를 찾기 어렵고 효율적인 활용이 어려운 문제를 해결합니다.

# Core Features  
- **텍스트 메모 작성**: 사용자가 직접 웹에서 텍스트 입력하여 메모 작성
- **음성 입력 (STT 변환)**: 마이크 입력을 실시간 텍스트로 변환하여 빠른 메모 작성
- **AI 요약**: 긴 메모 내용을 간결한 핵심 요약으로 제공하여 핵심 내용 파악 용이
- **자동 카테고리 분류**: 메모 내용을 주제 기반으로 자동 분류하여 체계적인 정리
- **검색 기능**: 키워드, 작성일, 카테고리별 검색으로 필요한 메모 빠른 찾기
- **데이터베이스 저장**: 모든 메모는 서버의 DB에 저장하여 안전한 보관
- **사용자 통계**: 작성 빈도, 사용 패턴 등 분석 정보 제공

# User Experience  
**주요 사용자:**
- 직장인/프리랜서: 회의나 작업 중 웹에서 빠르게 메모하고, 자동 정리되길 원함
- 대학생: 노트북을 통해 수업 내용을 음성/텍스트로 기록하고 정리하길 원함
- 일반 사용자: 웹에서 손쉽게 일상 메모를 작성하고 필요할 때 검색하길 원함

**핵심 사용자 흐름:**
1. 사용자가 웹 앱을 실행해 로그인
2. '새 메모' 버튼 클릭 → 음성 녹음 시작
3. STT 기술로 실시간 텍스트 변환
4. 메모 저장 시, 자동으로 AI 요약 및 카테고리 분류
5. 전체 메모와 요약본이 웹 화면에 동시 표시됨
6. 데이터는 자체 데이터베이스에 저장되어 웹에서 언제든지 확인 가능

# Technical Architecture  
**시스템 구성요소:**
- 프론트엔드: Next.js 기반 웹 애플리케이션
- 백엔드: API 서버 (Node.js/Express 또는 Next.js API Routes)
- 데이터베이스: PostgreSQL 또는 MongoDB
- AI 서비스: OpenAI API 또는 유사 서비스 (요약 및 분류)
- STT 서비스: Web Speech API 또는 외부 STT 서비스
- 인증: JWT 기반 사용자 인증

**데이터 모델:**
- 사용자(User): id, email, password, created_at
- 메모(Memo): id, user_id, title, content, summary, category, created_at, updated_at
- 카테고리(Category): id, name, description

**API 및 통합:**
- RESTful API 또는 GraphQL
- AI 서비스 API 통합 (요약, 분류)
- STT 서비스 API 통합
- 파일 업로드 API (음성 파일)

**인프라 요구사항:**
- 웹 서버 (Vercel, Netlify 등)
- 데이터베이스 서버
- AI 서비스 API 키 관리
- 환경 변수 관리

# Risks and Mitigations  
**기술적 도전:**
- STT 정확도: 여러 STT 서비스 테스트 및 선택
- AI API 비용: 사용량 기반 요금제 및 캐싱 전략
- 실시간 처리: 비동기 처리 및 로딩 상태 관리

**MVP 범위 조정:**
- 초기에는 텍스트 메모에 집중, 음성 기능은 2차 개발
- AI 기능은 기본 요약만 구현, 고급 분류는 후속 개발
- 검색 기능은 키워드 검색만 우선 구현

**리소스 제약:**
- 오픈소스 라이브러리 활용
- 클라우드 서비스 무료 티어 활용
- 단계적 기능 개발로 개발 시간 분산
